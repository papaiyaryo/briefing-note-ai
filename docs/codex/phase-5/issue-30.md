## Issue 概要

画像・OCR テキスト・Markdown の扱いが就活情報として安全かどうかを確認する。
本 Issue は **実装の追加ではなく確認作業が中心**(必要な修正は確認の結果に応じて行う)。

## スコープ

- ログ出力に画像内容や OCR 全文が含まれないか確認する
- API キーや `.env` がコミットされていないか確認する
- 外部送信なしの MVP 前提を README / docs に明記する

## スコープ外

- 本番監査体制
- 認証・権限管理

## 確認手順

1. **ログ出力の確認**

   ```bash
   grep -rn "console\.\(log\|error\|warn\|debug\)" src app
   ```

   現状(設計時点)では `src/` `app/` に `console.*` 呼び出しは存在しない。
   仮に見つかった場合は、画像内容・OCR 全文・生成 Markdown・個人情報を含むログでないかを確認し、
   含む場合は該当箇所を Issue の Scope 内で修正する。

2. **秘密情報・`.env` の確認**

   ```bash
   git ls-files | grep -E "^\.env($|\.)" 
   cat .gitignore | grep -n "\.env"
   cat .env.example
   ```

   - `.env` が追跡されていないこと(`.gitignore` に `.env` / `.env.*` / `!.env.example` があること)
   - `.env.example` の値がすべて空またはダミーであること
   - リポジトリ全体に実 API キーらしき文字列(`sk-`, `AIza` 等のプレフィックス)が
     含まれていないことを `git grep` で確認する。

3. **外部送信の有無の確認**

   - `src/lib/` に `fetch` / `axios` 等の外部通信コードがないことを確認する
     (Phase 5 時点では OpenAI / Google Drive 未接続のため、存在しないはずである)。
   - `app/api/health/route.ts` がアプリ内ヘルスチェックのみで、外部送信を行っていないことを確認する。

4. **README / docs への明記**

   - README に「MVP では外部 API に接続しない」「画像・OCR・Markdown はブラウザ内に
     保持され、サーバーに保存されない」ことを明記する一文を追加する
     (#31 の README 更新と重複しないよう、#31 で行う README 更新と合わせて 1 箇所に
     まとめてもよい。本 Issue では確認結果を残し、文言追加は #31 と調整する)。

## 実装ステップ

1. 上記 1〜3 の確認をすべて行い、結果(問題なし、または見つかった指摘事項)を
   PR 本文または Issue コメントに記録する。
2. 問題が見つかった場合のみ、Scope 内の最小修正を行う(例: 不要な `console.log` の削除)。
3. README または `docs/ci-policy.md` に「外部送信なし」の明記が既に十分か確認し、
   不足があれば 1 段落追記する(大規模な書き直しは #31 に譲る)。

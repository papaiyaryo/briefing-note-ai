# User Flow

## MVP の基本フロー

1. ユーザーがトップ画面を開く
2. 紙の企業説明会メモ画像をアップロードする
3. アップロード画像のプレビューを確認する
4. OCR を実行する
5. OCR 結果を確認する
6. OCR 結果をもとに Markdown を生成する
7. 生成された Markdown を編集する
8. 編集済み Markdown を `.md` としてダウンロードする

## 画面単位の想定

### 1. アップロード画面

目的は、ユーザーがメモ画像を選択し、処理を開始できるようにすることです。

主な要素:

- 画像アップロード領域
- 対応ファイル形式の表示
- 選択済み画像のプレビュー
- OCR 実行ボタン
- エラー表示

### 2. OCR 結果確認画面

目的は、画像から抽出されたテキストを確認し、Markdown 生成に進めるようにすることです。

主な要素:

- アップロード画像の縮小プレビュー
- OCR 結果テキスト
- OCR 再実行ボタン
- Markdown 生成ボタン
- OCR 失敗時のエラー表示

### 3. Markdown 編集画面

目的は、生成された企業研究メモをユーザーが自分用に整えられるようにすることです。

主な要素:

- Markdown エディタ
- 必要に応じたプレビュー
- ダウンロードボタン
- 生成や保存に関する状態表示

## エラー時のフロー

### 画像アップロード失敗

- 対応していない形式の場合、対応形式を表示する
- ファイルサイズが大きすぎる場合、上限と再選択を促す

### OCR 失敗

- OCR に失敗したことを表示する
- 画像が暗い、ぼやけている、文字が小さいなどの可能性を簡潔に伝える
- 再アップロードまたは再実行できるようにする

### Markdown 生成失敗

- 生成に失敗したことを表示する
- OCR 結果が空または短すぎる場合は、その理由を表示する
- 再実行できるようにする

## 将来的な拡張フロー

### Google Drive 保存

1. ユーザーが Google アカウントで認証する
2. 保存先フォルダを選択する
3. 編集済み Markdown を Google Drive に保存する
4. 保存完了後、Drive 上のファイルリンクを表示する

### Web 補足

1. Markdown 内の企業名を確認する
2. 公式サイトや採用ページから補足情報を取得する
3. 出典付きで候補を表示する
4. ユーザーが採用する補足情報を選ぶ
5. Markdown に追記する

### 企業比較

1. 複数の企業メモを選択する
2. 比較観点を選ぶ
3. 共通点、違い、志望理由への使いやすさを表示する
4. 比較結果を Markdown として出力する

### 面接前復習モード

1. 対象企業のメモを選ぶ
2. 面接前に見るべき要点を抽出する
3. 志望理由、逆質問、企業理解、自分との接点を短く表示する
4. 必要に応じて復習用 Markdown を出力する

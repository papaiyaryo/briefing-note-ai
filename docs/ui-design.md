# MVP UI デザイン

## 目的

MVP の 3 画面(アップロード / OCR 結果確認 / Markdown 編集)の画面構成・レイアウト・状態を定義し、Phase 3 の実装で迷わず参照できるようにする。

色・タイポグラフィ・余白・コンポーネントの個別ルールは `docs/ui-spec.md` のデザインシステム方針に従う。
画面の目的と要素は `docs/user-flow.md` の「画面単位の想定」に対応させる。

## Figma との関係

MVP UI デザインは Figma MCP 経由で Figma 上に作成済みである(記録方法は `docs/figma-mcp-setup.md` の「取得対象の管理方針」に従う)。
このドキュメントは引き続き MVP UI デザインの参照元(source of truth)とし、Figma デザインとこのドキュメントに差分が出た場合は、このドキュメントを更新して整合させる。

```text
Target nodes (page: MVP UI):
- Upload step: 2:2
- OCR result step: 2:3
- Markdown editor step: 2:4
- State variations (error / empty / loading): 2:5
```

## 全体構成

MVP は 1 ページのステップ型 UI とし、ページ遷移を伴うルーティングは追加しない。

```text
+--------------------------------------------------------------+
| ヘッダー: アプリ名(h1)+ 一言説明                           |
+--------------------------------------------------------------+
| ステップ表示: [1 アップロード] → [2 OCR 確認] → [3 編集]     |
+--------------------------------------------------------------+
|                                                              |
|  コンテンツカード(現在ステップの内容のみ表示)              |
|                                                              |
+--------------------------------------------------------------+
| フッター: 補足注記(任意・最小限)                           |
+--------------------------------------------------------------+
```

- ページ背景は `bg-slate-50`、コンテンツは白カード(`bg-white border border-slate-200 rounded-lg shadow-sm`)
- コンテンツ最大幅は `max-w-3xl`(Markdown 編集ステップのみ `max-w-5xl` まで可)
- ステップ表示は現在地表示であり、クリックでの自由遷移は提供しない
- 1 画面 1 目的とし、現在ステップ以外のフォームやボタンを同時に表示しない
- MVP 外機能(Drive 保存、Web 補足、企業比較、復習モード)への導線は配置しない

## 画面 1: アップロード

目的: メモ画像を 1 枚選択し、OCR を開始できるようにする。

```text
+------------------------------------------------------+
| ステップ表示: [1] アップロード(現在) 2  3          |
+------------------------------------------------------+
| h2: メモ画像をアップロード                           |
| 説明 1 文(text-sm text-slate-500)                  |
|                                                      |
| +--------------------------------------------------+ |
| |  破線ボックス(border-dashed)                   | |
| |  「ここに画像をドラッグ&ドロップ                | |
| |    またはファイルを選択」                        | |
| |  対応形式: JPG / PNG、サイズ上限の表示           | |
| |  [ファイルを選択](input type="file")           | |
| +--------------------------------------------------+ |
|                                                      |
| (選択済みのとき)                                   |
| +--------------------------------------------------+ |
| | 画像プレビュー(alt: アップロードしたメモ画像)  | |
| | ファイル名 + [別の画像を選択](Secondary)       | |
| +--------------------------------------------------+ |
|                                                      |
|                次へ: [OCR を実行する](Primary)     |
+------------------------------------------------------+
```

要素と状態:

| 状態 | 表示 |
| --- | --- |
| 未選択(空状態) | 破線ボックスに説明 1 文と対応形式・サイズ上限。Primary ボタンは disabled |
| ドラッグ中 | 破線ボックスの枠を `border-teal-700` に変更 |
| 選択済み | 画像プレビューとファイル名を表示し、Primary ボタンを有効化 |
| 形式・サイズエラー | エラーボックスに対応形式または上限を明示し、再選択を促す |
| OCR 実行中 | Primary ボタンを「OCR を実行しています…」表記 + disabled |

## 画面 2: OCR 結果確認

目的: 抽出テキストを確認し、Markdown 生成に進めるようにする。

```text
+------------------------------------------------------+
| ステップ表示: 1  [2] OCR 確認(現在) 3              |
+------------------------------------------------------+
| h2: OCR 結果を確認                                   |
|                                                      |
| +----------------+  +------------------------------+ |
| | 画像縮小       |  | OCR 結果テキスト             | |
| | プレビュー     |  | (textarea / font-mono)      | |
| | (読み取り元   |  |                              | |
| |  の参照用)    |  |                              | |
| +----------------+  +------------------------------+ |
|                                                      |
| [OCR を再実行](Secondary)                          |
|                [Markdown を生成する](Primary)      |
+------------------------------------------------------+
```

- 画像プレビューと OCR 結果は横並び(`md:` 以上)、狭い画面では縦積みにする
- OCR 結果はユーザーが手で修正できる `textarea` とする

要素と状態:

| 状態 | 表示 |
| --- | --- |
| OCR 成功 | 結果テキストを表示し、Primary ボタンを有効化 |
| OCR 失敗 | エラーボックスに失敗の旨と原因の可能性(暗い・ぼやけ・文字が小さい)を 1〜2 文で表示し、再実行・再アップロードへ誘導 |
| 結果が空・短すぎる | その旨を表示し、再実行または画像の再選択を促す |
| Markdown 生成中 | Primary ボタンを「Markdown を生成しています…」表記 + disabled |

## 画面 3: Markdown 編集・プレビュー

目的: 生成された企業メモを自分用に整え、`.md` としてダウンロードできるようにする。

```text
+----------------------------------------------------------------+
| ステップ表示: 1  2  [3] Markdown 編集(現在)                  |
+----------------------------------------------------------------+
| h2: Markdown を編集してダウンロード                            |
|                                                                |
| [編集] [プレビュー](タブ切り替え。md 以上では左右 2 ペイン可)|
| +------------------------------+ +---------------------------+ |
| | Markdown エディタ            | | プレビュー                | |
| | (textarea / font-mono)      | | (見出し・リスト整形表示) | |
| +------------------------------+ +---------------------------+ |
|                                                                |
| [OCR 確認に戻る](Secondary)                                  |
|                  [.md をダウンロード](Primary)               |
+----------------------------------------------------------------+
```

- 編集領域は `docs/output-format.md` の Markdown 構成(説明会の事実 / 自分の所感 / 後で聞くことの分離)を初期値として表示する
- このステップのみ `max-w-5xl` まで広げてよい
- プレビューは MVP では簡易表示でよい(シンタックスハイライト等は不要)

要素と状態:

| 状態 | 表示 |
| --- | --- |
| 生成直後 | 生成された Markdown をエディタに表示。ダウンロードは有効 |
| 編集中 | 入力内容を即時保持(明示的な保存ボタンは置かない) |
| 生成失敗 | エラーボックスに失敗の旨と理由(OCR 結果が空・短すぎる場合はそれを明示)を表示し、再実行へ誘導 |
| 内容が空 | ダウンロードボタンを disabled にし、注記を表示 |

## エラー・空状態・ローディングの共通方針

`docs/ui-spec.md` の「エラー・空状態・ローディング」に従う。

- エラーは `bg-red-50 border border-red-200 text-red-600` のボックスで、原因と次の操作を 1〜2 文で示す
- 空状態は説明 1 文 + 開始操作のみとし、イラストや長文を使わない
- ローディングは処理中テキスト + ボタン disabled とし、スケルトン UI は使わない
- エラー文言の内容は `docs/user-flow.md` の「エラー時のフロー」に対応させる

## レスポンシブ方針

- 基準はモバイル(縦積み・1 カラム)とし、`md:` 以上で横並びレイアウトを有効にする
- 画面 2 の画像/テキスト横並び、画面 3 の 2 ペイン表示は `md:` 以上のみ
- 横スクロールを発生させない

## スコープ外

- React / Next.js での実装(Phase 3)
- 実 OCR・実 LLM 処理(Phase 6)
- Google Drive 保存、Web 補足、企業比較、面接前復習モードの画面設計

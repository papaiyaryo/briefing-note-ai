# Web 補足機能 設計

## 目的

Web 補足機能は、説明会や面談で得た紙メモ由来の情報を主情報として保ちつつ、企業公式サイトや採用ページから確認できる補足情報を、出典付きで後から参照できるようにするための Post-MVP 機能である。

この機能は企業情報を自動で確定させるものではない。ユーザーが確認し、採用した情報だけを Markdown の `## Web 補足情報` セクションへ追加する。

## MVP との境界

Web 補足機能は MVP の対象外とする。

MVP では次の状態を維持する。

- Web ページ取得、検索、スクレイピングを実装しない
- Web 由来情報を自動生成 Markdown に混ぜない
- `## Web 補足情報` は空、または `未使用` として扱う
- 企業情報を LLM の推測で追加しない

Phase 8 以降で実装する場合も、紙メモ由来情報と Web 由来情報の分離を崩さない。

## 情報源の分離方針

紙メモ由来情報と Web 由来情報は、次のように扱いを分ける。

| 区分 | 主な入力 | Markdown 上の扱い | 信頼の前提 |
| --- | --- | --- | --- |
| 紙メモ由来情報 | OCR テキスト、ユーザー入力、説明会メモ | 事実、強調点、印象、質問、ES・面接材料の各セクション | ユーザーが説明会等で得た一次メモ |
| Web 由来情報 | 公式採用ページ、企業公式サイト、IR・プレスリリース | `## Web 補足情報` セクションのみ | 出典と取得日に基づく補足情報 |

禁止事項:

- Web 由来情報を「説明会で聞いた事実」として書かない
- 紙メモ由来の個人的な印象を Web 情報で上書きしない
- Web 情報を出典なしで Markdown に入れない
- LLM の推測だけで企業情報を補完しない
- Web 情報と紙メモ情報が矛盾する場合に自動で片方を正としない

矛盾がある場合は、Web 補足側に `要確認` として残し、ユーザー判断に委ねる。

## 取得対象と優先度

Web 補足で参照する情報源は、企業本人または企業が公式に公開している情報を優先する。

| 優先度 | 情報源 | 用途 |
| --- | --- | --- |
| 1 | 企業の公式採用ページ、キャリアページ | 採用メッセージ、募集職種、選考情報、求める人物像の確認 |
| 2 | 企業公式サイトの会社概要、事業紹介 | 事業内容、拠点、沿革、サービス概要の確認 |
| 3 | 企業公式の IR、プレスリリース、ニュースリリース | 最新の事業発表、業績・方針の確認 |
| 4 | 官公庁、証券取引所など公的・準公的情報 | 公式情報の補助確認 |

原則として対象外にする情報源:

- SNS 投稿
- 口コミサイト
- 転職・就活メディアの記事
- 個人ブログ
- 出典不明のまとめ記事
- 一般ニュース記事のみを根拠にした企業評価

対象外情報を扱う必要がある場合は、Phase 8 の別 Issue で明示的に仕様化する。

## 出典メタデータ仕様

Web 補足候補は、本文だけでなく出典メタデータを必ず保持する。

```ts
type WebSupplementSource = {
  url: string;
  title: string;
  retrievedAt: string; // ISO 8601形式
  sourceType: "official-recruit" | "official-company" | "ir" | "press-release" | "public-source";
  confidence: "confirmed" | "unverified";
};
```

Markdown に採用する場合は、最低限次の情報を残す。

- 補足内容
- 出典 URL
- 取得日
- 情報源種別
- 確認状態（確認済み / 要確認）

出力例:

```md
## Web 補足情報

- 公式採用ページでは、職種別採用情報が公開されている。
  - 出典: https://example.com/recruit
  - 取得日: 2026-07-01
  - 情報源: 公式採用ページ
  - 確認状態: 確認済み
```

## ユーザー確認フロー

Web 補足は、生成 Markdown に自動統合しない。次の確認フローを必須とする。

1. Markdown 生成後、ユーザーが任意で「Web 補足を追加」を選択する
2. システムが補足候補を取得または生成する
3. 候補一覧に、補足内容・出典 URL・取得日・情報源種別・確認状態を表示する
4. ユーザーが候補ごとに「採用」「却下」「要確認のまま保持」を選ぶ
5. 採用または保持された候補だけを `## Web 補足情報` に追記する
6. 紙メモ由来セクションには追記しない

この流れにより、Web 情報がユーザーの説明会メモや個人的な印象を自動で上書きしないようにする。

## LLM 利用時のルール

LLM を使って Web 補足候補を要約する場合は、次のルールを守る。

- 入力に含まれる出典テキストまたは取得済みページ内容だけを根拠にする
- 企業情報を推測で追加しない
- 出典 URL と取得日を要約結果に紐づける
- 紙メモ由来情報と Web 由来情報を混ぜない
- 不明点や矛盾がある場合は `要確認` と明示する
- 可能であれば Markdown 生成前に構造化 JSON として検証する

## 後続 Issue への引き継ぎ

Issue #42 以降で実装する場合は、この設計を前提に次を分けて扱う。

- Web 情報の取得・要約ロジック
- 出典メタデータを含む型定義
- ユーザー確認 UI
- 採用済み Web 補足の Markdown 統合

この Issue では、Web スクレイピング実装や自動企業データベース作成は行わない。

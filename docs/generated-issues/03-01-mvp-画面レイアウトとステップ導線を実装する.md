# [Phase 3] MVP 画面レイアウトとステップ導線を実装する

## Phase

Phase 3: MVP UI

## Priority

High

## Labels

- phase:3
- mvp
- frontend

## Goal

アップロード、OCR確認、Markdown編集の流れを一画面または明確なステップで操作できるようにする。

## Scope

- MVP の画面骨格を実装する
- 現在の処理ステップを表示する
- 次に行う操作が分かる導線を作る

## Out of Scope

- OCR / Markdown 生成ロジック
- 外部 API 連携

## Acceptance Criteria

- [ ] ユーザーが現在位置と次の操作を把握できる
- [ ] モバイルとデスクトップで破綻しない
- [ ] MVP 外の機能導線が表示されない

## Dependencies

Depends on Phase 2 UI spec and Phase 1 setup。

## Notes

実データ処理は Phase 4 で接続する。

# [Phase 5] E2E テスト方針を整理する

## Phase

Phase 5: MVP品質改善・テスト

## Priority

Medium

## Labels

- phase:5
- mvp
- test
- docs

## Goal

MVP のアップロードからダウンロードまでを将来 E2E で確認する方針を決める。

## Scope

- E2E 対象フローを整理する
- Playwright 等の候補を比較する
- 外部 API なしで実行する方針を決める

## Out of Scope

- E2E テスト実装
- 外部 API を使うテスト

## Acceptance Criteria

- [ ] E2E の対象フローが docs に記録されている
- [ ] 導入タイミングが明確である
- [ ] 秘密情報を使わないテスト方針になっている

## Dependencies

Depends on Phase 4 MVP flow。

## Notes

MVP では方針整理まででも可。

# [Phase 3] エラー・空状態・ローディング状態を実装する

## Phase

Phase 3: MVP UI

## Priority

Medium

## Labels

- phase:3
- mvp
- frontend

## Goal

ユーザーが処理中や失敗時に次の行動を理解できる状態表示を整える。

## Scope

- アップロード未選択の空状態を実装する
- OCR / Markdown 生成中のローディングを実装する
- 失敗時のエラー表示と再試行導線を実装する

## Out of Scope

- 外部 API の詳細エラー分類
- 監視ログ基盤

## Acceptance Criteria

- [ ] 各ステップに空状態がある
- [ ] 処理中に二重実行しにくい
- [ ] エラー文言が次の行動を示している

## Dependencies

Depends on Phase 3 UI components。

## Notes

個人情報を含む OCR 全文をエラーに出さない。

# Requirements

## スコープ

このドキュメントは、MVP の機能要件と非機能要件を定義します。現時点では React 実装や具体的な UI 実装は行わず、設計の前提を整理します。

## MVP 機能要件

### 画像アップロード

- ユーザーは紙の企業説明会メモを撮影した画像をアップロードできる
- 対応形式は初期段階では `png`, `jpg`, `jpeg`, `webp` を想定する
- 1 回の処理では、まず 1 画像を対象にする
- アップロード後、画像のファイル名とプレビューを確認できる

### OCR 結果表示

- アップロード画像から OCR テキストを抽出する
- OCR 結果はユーザーが確認できる形で表示する
- OCR 結果は Markdown 生成前に編集可能にするか、少なくともコピー可能にする
- OCR が失敗した場合は、失敗理由と再試行できる導線を表示する

### Markdown 生成

- OCR 結果をもとに、企業研究向けの Markdown を生成する
- 生成 Markdown は以下のような構造を基本とする

```markdown
# 企業名

## 説明会概要

## 事業内容

## 強み・特徴

## 求める人物像

## 気になった点

## ES・面接で使えそうな材料

## 次に調べること
```

- OCR 結果に存在しない情報は、断定せず「不明」または「要確認」として扱う
- LLM を使う場合は、推測と事実を混同しないプロンプト設計にする

### Markdown 編集

- ユーザーは生成された Markdown を編集できる
- 編集中の内容は画面上で保持される
- 初期 MVP ではサーバー保存を必須にしない

### `.md` ダウンロード

- ユーザーは編集済み Markdown を `.md` ファイルとしてダウンロードできる
- ファイル名は企業名が取得できる場合は企業名を含める
- 企業名が不明な場合は日時ベースのファイル名にする

## 将来的な機能要件

### OpenAI Vision API

- 画像から直接、文字情報と構造情報を抽出する
- OCR だけでは読み取りにくい表、矢印、囲み、強調表現を補助的に解釈する

### Google Drive 保存

- 生成・編集した Markdown を Google Drive に保存する
- ユーザーごとの認証と保存先フォルダ選択を扱う

### Web 補足

- 企業名をもとに、公式サイトや採用情報から補足情報を取得する
- 補足情報は出典付きで Markdown に追記する
- MVP では実装しない

### 企業比較

- 複数企業の Markdown を横断して比較する
- 事業内容、志望理由、求める人物像、懸念点などの観点で比較できる

### 面接前復習モード

- 面接直前に確認すべき要点を短く表示する
- 「志望理由」「逆質問」「企業理解」「自分との接点」を中心に再整理する

## 非機能要件

- ユーザーのメモ画像や OCR テキストは個人情報を含む可能性があるため、不要に外部送信しない
- 外部 API に送信する場合は、送信対象と目的を明示できる設計にする
- MVP では少ない操作で完了できることを優先する
- 画像処理や LLM 処理中は、処理中であることが分かる表示を用意する
- エラー時には、ユーザーが次に何をすればよいか分かる文言を表示する

## MVP でやらないこと

- ユーザーアカウント管理
- 複数端末同期
- Google Drive 連携
- Web 検索による企業情報補足
- 複数企業比較
- 面接前復習モード
- 本格的な権限管理
